# 3D Viewer — вьювер-мост для инженеров

Веб-приложение для просмотра и согласования **чертежей (PDF)** и **3D-моделей** в браузере: без AutoCAD и КОМПАС-3D у получателя. Поддерживаются **аннотированные скриншоты** и **PDF-отчёт** для фиксации замечаний.

**Версия:** 1.4.0 (см. `viewer/package.json`)

---

## Зачем это нужно

На рынке есть **PDF-редакторы** (общие документы, не заточены под инженерные чертежи и отчёты из скриншотов) и **CAD-просмотрщики** (файлы CAD, а не сценарий «ссылка в чат → посмотрел в браузере → скриншот с подписью»).

Этот проект занимает нишу **«инженер-мост»**:

- простой просмотр **PDF + опционально 3D**;
- **аннотации** на скриншотах и **сборка отчёта в один PDF**;
- при работе с **STEP** — вкладка **«Спецификация»** (таблица по ГОСТ) через локальный сервер.

3D — **усиление** к PDF, а не обязательный слой: фронт можно запускать без сервера (тогда нет спецификации и серверной конвертации STEP, но PDF и 3D по-прежнему доступны).

## Кому полезно

| Роль | Задача |
|------|--------|
| **Конструкторы, проектировщики, технологи** | Показать чертёж или узел, сделать скриншот с подписью/стрелкой, собрать **отчёт-PDF**. |
| **Монтаж, производство, заказчик** | Открыть ссылку в браузере, увидеть **PDF** и при необходимости **3D** без установки CAD. |
| **Команды в Teams / Telegram** | Вместо «скинь файл» — **одна страница** (чертёж + модель), далее скриншоты и ссылки в чат. |

**Пример сценария (например, фасады / профили):** по ссылке или QR открывается страница вьювера — **PDF чертежа**, при необходимости **3D**, скриншот с аннотацией → в чат или в отчёт.

## Что уже есть в коде (факты)

### Режимы и вкладки

- **3D / 2D** — одна вкладка: 3D-сцена и PDF (чертежи).
- **Спецификация** — отдельная вкладка: таблица по **ГОСТ** (Поз., Обозначение, Наименование, Кол., Масса ед./общ., Примечание). Данные приходят с **локального FastAPI** после загрузки **STEP** (метаданные из файла + билдер спецификации).

### PDF и отчёт

- Просмотр **PDF** (PDF.js).
- **Скриншоты 2D/3D**, редактор с фигурами и текстом, добавление в **PDF-отчёт** (jspdf): шапка по первому листу PDF, скриншоты, блок измерений.

### 3D

- Форматы: **STL**, **STEP** (.stp, .step), **IGES** (.igs, .iges), **GLB/GLTF**.
- **STEP/IGES:** сначала пробуется конвертация на **сервере** (CadQuery + trimesh → GLB); при недоступности сервера — **opencascade.js (WASM)** в браузере.
- **Лимит загрузки STEP/IGES:** **30 МБ** (больше — загрузка не выполняется, предупреждение в лог).
- **Камера:** TrackballControls; меню **ориентаций**; **«Перпендикулярно»** — клик по грани, затем кнопка (кнопка всегда активна, без грани — подсказка в лог).
- **Каркас** — не проволока, а **полупрозрачные грани**; **opacity** 0.05–0.95 (поле, слайдер, колёсико мыши).
- **Сетка пола** (GridHelper) с кнопкой вкл/выкл; размер сетки адаптируется к габаритам модели.
- **Оттенок модели**: быстрый регулятор тона (светлее/темнее) без тяжёлых материалов и теней.
- **Сечение** — клик по грани, смещение плоскости, фиксация / снятие.
- **Измерения** — два клика, привязка к геометрии (пересечение, вершина, центр грани, ребро); расстояние и ΔX/Y/Z в мм.
- **Журнал измерений** — плавающее окно поверх канваса в виде таблицы (колонки/строки): номер, длина, перпендикуляр, параметры треугольника; по клику строка восстанавливает точки/линии в сцене, в конце строки есть удаление измерения.
- **Метрики сцены** — расчёт и показ площади/объёма/числа треугольников для моделей в сцене.
- **Экспорт:** GLB, STL.

### Производительность и логирование

- Для тяжёлых сборок оптимизирован режим `distance`: убраны дорогие обходы топологии из hover/click, добавлен throttling hover-пересчётов.
- Во время вращения камеры включается упрощённый рендер (снижение `pixelRatio`) и отключаются лишние вычисления UI-лейблов измерений.
- **Режимы логов:** `Полный` (debug), `Обычный` (info), `Тихий` (warn+error).

### Стек

- **Vue 3**, **TypeScript**, **Vite**, **Three.js**
- **opencascade.js** — STEP/IGES в браузере
- **FastAPI** (`server/`) — метаданные STEP, спецификация, STEP→GLB
- **PDF.js**, **jspdf**

## Запуск

### Всё сразу (сервер + фронт)

```batch
run.bat
```

Два окна: сервер **:8000**, Vite **http://localhost:5173**.

### Только фронт

```bash
cd viewer
npm install
npm run dev
```

Без сервера: **нет** вкладки спецификации по STEP и **нет** серверной конвертации STEP; STEP до **30 МБ** — через WASM в браузере.

### Сервер отдельно

```powershell
cd server
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

API и детали — [server/README.md](server/README.md).

## Сборка

```bash
cd viewer
npm run build
```

Артефакты: `viewer/dist/`.

## Структура репозитория

| Путь | Назначение |
|------|------------|
| `viewer/` | Фронт: `App.vue` (режимы, отчёт), `Viewer3D.vue` (сцена, PDF-вкладка, спецификация, каркас, сечение, измерения), `stepLoader.ts` |
| `server/` | FastAPI: `/api/step/metadata`, `/api/convert/step-to-glb` |
| `spec_src/` | Исходники логики спецификации и материалов (ориентир для сервера) |

## Планы (дорожная карта)

- **PWA** — установка с домашнего экрана, удобнее по ссылке/QR без «магазина приложений».
- **Microsoft Teams и др.** — как канал доставки: вкладка или сообщение со **ссылкой** на задеплоенный вьювер (отдельная «интеграция SDK» не обязательна на первом этапе).
- **Яндекс.Диск (или аналог)** — папка проекта, файлы по QR/ссылке, единая точка хранения рядом с вьювером.
