# 3D Viewer — веб-инструмент для чертежей и 3D

Веб-приложение для просмотра и согласования **PDF-чертежей** и **3D-моделей** в браузере: получателю не нужны AutoCAD, КОМПАС-3D и т.п. Поддерживаются **аннотированные скриншоты**, **PDF-отчёт** с измерениями, **мультимодельная сборка** с сохраняемыми сопряжениями по плоскостям и вкладка **«Спецификация»** по STEP (через локальный сервер).

**Версия фронта:** 1.4.0 (см. `viewer/package.json`)

---

## Зачем это нужно

На рынке обычно разделены:

- **PDF-редакторы** — не заточены под инженерный цикл «открыл чертёж → отметил на 3D → в отчёт»;
- **CAD-просмотрщики** — отдельные файлы и лицензии, редко единый сценарий «ссылка в чат → браузер → скриншот с подписью → один PDF».

Этот проект — **«инженерный мост»** в одной странице:

| Возможность | Описание |
|-------------|----------|
| **PDF + 3D** | Одна вкладка: чертёж и модель рядом. |
| **Скриншоты и отчёт** | Редактор фигур/текста, сборка **одного PDF** (jspdf), блок измерений. |
| **Сборка** | До нескольких моделей в сцене, **связи по плоскостям** (совпадение, расстояние, симметрия по ширине), повторное применение после перетаскивания. |
| **Спецификация** | Таблица по **ГОСТ** после загрузки **STEP** через **FastAPI** (метаданные + билдер). |

3D — **усиление** к PDF, а не обязательный слой: фронт можно запускать **без сервера** (тогда нет серверной конвертации STEP и вкладки спецификации по API; STEP всё равно можно грузить через **WASM** в браузере в пределах лимита размера).

## Возможности 3D-сцены (`Viewer3D.vue`)

### Форматы и загрузка

- **STL**, **STEP** (`.stp`, `.step`), **IGES** (`.igs`, `.iges`), **GLB/GLTF**.
- **STEP/IGES:** сначала запрос **серверной** конвертации (CadQuery + trimesh → GLB); при недоступности или ошибке — **opencascade.js (WASM)** в браузере.
- **Лимит STEP/IGES на фронте:** **30 МБ** (больше — загрузка не выполняется, сообщение в лог).
- На сервере отдельный лимит для конвертации — см. [server/README.md](server/README.md).

### Навигация и вид

- **TrackballControls**, меню **ориентаций**, режим **«Перпендикулярно к грани»** (клик по грани + кнопка).
- **Сетка пола** (вкл/выкл), размер от габаритов модели.
- Во время вращения камеры — снижение `pixelRatio` и throttling тяжёлых расчётов (hover и т.д.).

### Материалы и освещение

- Режим **«С освещением» (lit):** **MeshPhongMaterial**, направленные источники + ambient/hemisphere; **карты теней** (мягкие PCF) от ключевого света, подстройка теневой камеры под **габариты сцены**; баланс fill-света, чтобы тени читались на **светлой** модели.
- Режим **«Без шейдинга» (unlit):** уплощённый вид (Lambert + emissive), **тени отключены** для скорости и предсказуемого вида.
- **Оттенок / яркость модели** — быстрый слайдер без смены типа материала.
- **Каркас:** не проволока, а **полупрозрачные грани**; прозрачность 0.05–0.95 (поле, слайдер, колёсико).

### Сечение и измерения

- **Сечение:** выбор оси/смещение, клик по грани для ориентации плоскости, фиксация/снятие.
- **Измерения:** два клика, привязка (грань, вершина, ребро и др.); длина и ΔX/ΔY/ΔZ в мм.
- **Журнал измерений** — таблица поверх канваса: восстановление по клику, удаление строки, столбцы для перпендикуляра/треугольника где применимо.

### Сборка (несколько моделей)

- Выбор **источника** и **опорной** модели из списка загруженных **в сцене**.
- Типы сопряжений: **плоскость ↔ плоскость**, **по расстоянию**, **симметрия по ширине** (четыре плоскости: две на базе, две на детали).
- **Пошаговый выбор граней** подсвечивается в сцене (треугольник грани или маркер плоскости).
- После **«Применить сопряжение»** связь сохраняется в таблице; при **перетаскивании** детали связи **пересчитываются**.
- Таблица **«Зафиксированные связи»:** клик по строке подсвечивает плоскости этой связи в 3D; повторный клик снимает подсветку; удаление связи — кнопка в строке.

### Прочее в 3D

- **Метрики сцены:** площадь поверхности, объём, число треугольников (для моделей в сцене).
- **Экспорт:** GLB, STL.

### PDF (вкладка рядом с 3D)

- Просмотр через **PDF.js**.
- Скриншоты области 2D/3D, редактор аннотаций, включение в **итоговый PDF** с шапкой и блоком измерений.

### Спецификация (отдельная вкладка)

- Таблица по **ГОСТ:** позиция, обозначение, наименование, количество, масса ед./общ., примечание и др. по логике билдера.
- Данные приходят с **локального API** после анализа **STEP** (см. `server/`).

### Логирование

- Режимы: **полный** (debug), **обычный** (info), **тихий** (warn+error).

## Запуск

### Всё сразу (Windows)

Из корня репозитория:

```batch
run.bat
```

Ожидаемо: сервер на **:8000**, Vite на **http://localhost:5173** (два процесса/окна — см. ваш `run.bat`).

### Только фронт

```bash
cd viewer
npm install
npm run dev
```

Без сервера: нет серверной конвертации STEP и нет данных спецификации с API; STEP до лимита — через WASM в браузере.

### Только сервер

```powershell
cd server
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

Health: `http://localhost:8000/api/health`  
Подробности API: [server/README.md](server/README.md).

## Структура репозитория

| Путь | Назначение |
|------|------------|
| `viewer/` | Vue-приложение: `App.vue` (вкладки, отчёт), `components/Viewer3D.vue` (сцена, PDF, сборка, измерения, сечение), `lib/stepLoader.ts` и др. |
| `server/` | FastAPI: метаданные STEP, конвертация STEP→GLB, эндпоинты для спецификации |
| `run.bat` | Точка входа «поднять всё локально» (если присутствует в вашей копии) |
| `spec_src/` | *(опционально в клоне)* отдельные исходники/прототипы вокруг спецификации и КОМПАС — не обязательны для сборки `viewer`+`server` |

Каталог **`icons/`** можно использовать для ресурсов интерфейса; в репозиторий имеет смысл класть только нужные файлы без тяжёлых тестовых наборов.

## Ограничения и ожидания

- Проект ориентирован на **инженерный обмен** и **внутреннее** использование; публичный коммерческий релиз требует отдельной проработки лицензий (OCCT, PDF.js, шрифты и т.д.).
- Тяжёлые сборки STEP: разумные лимиты размера файла, терпение при первой WASM-загрузке, по возможности — серверная конвертация для ускорения.
