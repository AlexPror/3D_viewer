# Анализ формата JT (Jupiter Tesselation) и конвертации в GLB

Документ сохраняет итоги исследования поддержки JT в проекте 3D Viewer. Функционал JT из приложения удалён; при необходимости его можно вернуть по этому анализу.

---

## 1. Почему JT рассматривался

JT — один из самых лёгких форматов для 3D-визуализации (сжатая тесселяция, LOD). Конвертировать JT в тяжёлые форматы (STL, OBJ) смысла нет — теряется выигрыш по размеру. Целевой формат — только GLB (бинарный glTF).

---

## 2. Что пробовали в проекте

- **Assimp:** в проекте был сервер (Node + Assimp CLI) для конвертации JT → GLB. В исходниках Assimp 6.0.4 **нет импортера JT** (в `ImporterRegistry.cpp` и `AssetLib/` формата JT нет). При вызове `assimp export file.jt out.glb` Assimp подставлял другой парсер (FBX и др.) и падал с ошибкой разбора. Вывод: Assimp для JT не подходит.

---

## 3. Спеки и стандарты JT

- **ISO 14306** — стандарт формата JT (актуальная версия 2017, готовится Part 4).
- **JT Open Program (Siemens):** https://plm.sw.siemens.com/en-US/plm-components/jt/jt-open-program — можно запросить JT File Format Reference (например, v10.6).
- **prostep / PSI 14** — детальное описание структуры (тесселяция, B-Rep, PMI); документы по подписке.

---

## 4. Готовые открытые решения (на момент анализа)

| Проект        | Язык  | Лицензия | Назначение                          |
|---------------|-------|----------|-------------------------------------|
| **JTConvert** | C#    | LGPL-3.0 | CLI и библиотека, чтение/конвертация JT |
| **oce-jt**    | C++   | GPL-2.0  | JT Reader, JTAssistant (форк Open CASCADE) |
| **FreeCAD JtReader** | C++ | LGPL | Чтение JT, экспорт в STL (не в GLB напрямую) |
| **JT-file-parser** | Java + Python | — | Парсинг JT, импорт в Blender |
| **jt-java** (corubolo) | Java | — | Парсинг JT, Logical Scene Graph (учебный) |
| **dxjt_toolkit** | C++ | — | Чтение JT |

Ни один из них не даёт «из коробки» стабильный JT → GLB без тяжёлых промежуточных форматов или без коммерческой лицензии.

---

## 5. Коммерческие варианты

- **CAD Exchanger Batch** — CLI, JT в т.ч. в GLB. На момент анализа продажи временно приостановлены, предлагают 3DViewStation.
- **3DViewStation / Kisters Automation Server (KAS)** — конвертация JT (в т.ч. в GLTF). Платно (~$1600+), есть 30-дневный триал.
- **JT Open Toolkit (JTTK)** — официальный C++ API от Siemens для чтения/записи JT (коммерческий).

---

## 6. Варианты «своего» конвертера

- **Прямой JT → GLB:** искать/использовать только те инструменты, которые отдают glTF/GLB.
- **Парсер JT → меш в памяти → GLB:** взять открытый парсер (oce-jt, FreeCAD, JTConvert и т.д.), получать меш в коде и писать только GLB (tinygltf, pygltflib и т.п.), без сохранения STL/OBJ.
- **Минимальный свой парсер по JT Open:** опираться на ISO 14306 и JT Open Reference, реализовать чтение только тесселяции для подмножества формата — объёмная разработка (недели).

---

## 7. Итог по проекту

Решено не поддерживать JT в текущем приложении. Весь JT-функционал (сервер конвертации, прокси `/api`, обработка .jt во вьюере) удалён. STEP/IGES по-прежнему обрабатываются в браузере через opencascade.js с конвертацией сразу в GLB.

При необходимости вернуть JT — использовать этот файл и перечисленные выше варианты (коммерческие конвертеры или доработка открытых парсеров с выводом в GLB).
