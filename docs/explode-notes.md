# Разнесение сборки (explode) — анализ и варианты на будущее

Функция разнесения была убрана из вьювера. Ниже — что было, почему не совпадало с eDrawings и что можно сделать в будущем.

---

## Что было реализовано

- **Группировка «деталей»** по связности мешей: два меша считались одной деталью, если у них находились общие вершины в мировых координатах (с точностью `VERTEX_EPS`). Использовались spatial hash вершин и Union-Find, сэмплирование до ~800 вершин на меш.
- **Направление разнесения:** от центра сборки к центру каждой группы.
- **Слайдер 0–100%** задавал величину смещения; позиции мешей пересчитывались в локальных координатах родителя.

Ограничение: структура сборки из STEP (дерево изделия) **не использовалась** — при загрузке вызывался `STEPControl_Reader` → `OneShape()`, в GLB попадала по сути одна форма/узел.

---

## Почему результат отличался от eDrawings

- **eDrawings** опирается на **структуру сборки из файла** (STEP AP 242/214): дерево продукт → компоненты → детали. Одна запись в дереве = одна группа при разнесении.
- В нашем случае «детали» определялись **только геометрией**: общие вершины = одна группа. Поэтому:
  - две детали, плотно прилегающие в модели, могли склеиваться в одну группу;
  - одна деталь, разбитая на несколько мешей, могла разъезжаться на несколько групп.

Пересохранение STEP через SolidWorks при текущей схеме (один shape) не меняет логику — структура сборки по-прежнему не используется.

---

## Что можно сделать в будущем

### 1. Разнесение по структуре сборки (как в eDrawings)

- Читать STEP в **XCAF-документ** (`STEPCAFControl_Reader` → `Transfer_1` в `TDocStd_Document`), а не снимать один shape.
- Экспортировать в GLB **этот документ** через `RWGltf_CafWriter`, чтобы в GLB было **несколько узлов** по числу компонентов/деталей.
- Во вьювере разносить **по узлам дерева сцены** (например, по родительским объектам из GLB), а не по связности мешей.

Ранее при включении XCAF появлялись чёрные формы — значит, нужно поправить **тесселяцию/экспорт** (обход всех меток, включая референсы, и корректная запись в GLB), а не отказываться от структуры.

### 2. Гибрид: структура + геометрия

- Если в файле есть дерево (XCAF) — разносить по нему.
- Если дерева нет (один shape, один узел в GLB) — оставить разнесение по связности мешей как запасной вариант.

### 3. Улучшение группировки по геометрии (без XCAF)

- Увеличить сэмплирование вершин или использовать более точное определение «общих» вершин (нормали, рёбра).
- Учитывать иерархию объектов в сцене (родительские узлы из GLB), если в одном узле несколько мешей — считать их одной деталью.

### 4. Направление разнесения

- Сейчас: от центра сборки к центру каждой группы. Можно добавить варианты: по осям X/Y/Z, по нормали к плоскости сборки, или сохранять направления из CAD, если они появятся в данных.

### 5. Список деталей и подсветка

- При возврате разнесения по структуре — снова вывести список деталей (из имён узлов/меток) с группировкой повторяющихся по имени (например, «Заклепка — 10 шт.») и подсветкой выбранной группы.

---

## Итог

Чтобы разнесение вело себя как в eDrawings, нужна **структура сборки из STEP** (XCAF → GLB с узлами по компонентам) и разнесение по этим узлам. Текущее удаление разнесения упрощает вьювер; при необходимости восстановления ориентироваться на этот документ и вариант 1 выше.
