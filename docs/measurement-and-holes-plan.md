## 1. Панорамирование (panRay) — сделано

- **Было:** `panRay.setFromCamera` — у `THREE.Ray` нет такого метода.
- **Сделано:** Заменён `panRay` на `panRaycaster`; в `onCanvasMouseMovePan` используется `panRaycaster.setFromCamera(...)` и `panRaycaster.ray.direction` для обновления камеры.

## 2. Подсветка граней после первого измерения

- **Проблема:** После первого измерения (2 точки) при следующих измерениях может переставать работать подсветка плоскостей (объединение треугольников в плоскость).
- **План:** Убедиться, что блок подсветки в `animate()` не зависит от `measurementPoints.length`. При воспроизведении — добавить временный лог и проверить выполнение блока. При индексированной геометрии проверить индексы в fallback-треугольнике подсветки. Опционально: автоочистка при начале нового измерения.

---

## 3. Четвёртая линия (перпендикуляр) — улучшено

- **Сделано:** При втором клике «в пустоту» используется нормаль последнего наведения (`lastHoverNormal`) или нормаль первой точки; перпендикуляр и подпись рисуются, если есть хотя бы одна из этих нормалей.
- **Хранение:** В `animate()` при наведении сохраняются `lastHoverNormal` и `lastHoverPoint`; при уходе курсора с модели — сброс в `null`.

## 4. Треугольник измерений

- Разложение по осям в фиксированной плоскости `MEASURE_PLANE_NORMAL (0, 1, 0)`. Без изменений.

## 5. Отверстия: распознавание по мешу — сделано

- **Сделано:** В `isCylinderAHole`: луч от центра окружности (`origin = center`), направление `-normal`; допустимый диапазон расстояния: `0.1*radius … 20*radius`, `raycaster.far = radius*25`. Так распознаётся больше отверстий (сквозные и неглубокие).

## 6. Отверстия и цвет: извлечение из STEP (JSON) и передача во вьюер

### 6.1. Отверстия из STEP

- **stepLoader.ts:** После `TransferRoots` и получения `shape` обойти грани (например `TopExp_Explorer` по `TopAbs_FACE`). Для каждой грани определить тип поверхности; для цилиндров — ось, радиус, центр. Собрать массив `holeFeatures` (например `{ axisPoint, axisDirection, radius }` в мировой СК).
- **Возврат:** Из `loadStepOrIgesToGlbUrl` возвращать не только GLB URL, а объект вида `{ glbUrl, holeFeatures }` (или второй аргумент).
- **Вьюер:** При загрузке STEP сохранять `holeFeatures`; при наведении/клике в режимах «Диаметр» и «Расст. между центрами отверстий» сопоставлять грань с записью из `holeFeatures` (по геометрии) и подставлять центр/радиус из JSON, если по мешу отверстие не распознано.

### 6.2. Цвет из STEP

- **stepLoader.ts:** При конвертации STEP в GLB с использованием OpenCascade:
  - Проверить, экспортирует ли `RWGltf_CafWriter` цвета в GLB (материалы/вершины). Если да — дополнительная выдача цветов в JSON может не понадобиться.
  - Если цвета в GLB не попадают: при обходе граней/тел читать цвет (например `XCAFDoc_ColorTool::GetColor`) в RGB. Собрать массив `colorByFaceId` (или по индексу/имени объекта): `{ id, r, g, b }`.
  - Возвращать вместе с GLB: `{ glbUrl, holeFeatures?, colorByFaceId? }`.
- **Вьюер:** При загрузке GLB, полученного из STEP, если пришёл `colorByFaceId`, сопоставить меши/узлы сценария с id из STEP и задать материалы/цвета (`mesh.material.color` и т.д.). Если цвета уже в GLB — не перезаписывать материал.
- **Совместимость:** Для IGES/STL/других форматов оставить текущее отображение (без цветов из STEP).
